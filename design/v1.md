# UI Design Views Skill

Diseña vistas HTML hermosas, modernas y cohesivas. Sin lógica de backend, sin estados complejos —
solo diseño visual de primer nivel, listo para usar como prototipo o entrega final de UI.

---

## Paso 1: Leer el contexto y elegir el stack visual

Antes de escribir una sola línea, identifica el tipo de producto y asigna el stack correcto:

| Tipo de producto | Stack recomendado |
|---|---|
| Dashboard / admin / SaaS | Tailwind CDN + DaisyUI o Flowbite + Lucide Icons + AOS |
| Landing page / startup | Tailwind CDN + AOS + GSAP para hero + Lucide Icons |
| Portfolio / agencia creativa | Tailwind CDN + GSAP + fuente display llamativa |
| App móvil-style (PWA visual) | Tailwind CDN + DaisyUI + bottom nav + max-w-md |
| E-commerce visual | Tailwind CDN + Flowbite + grid de productos |
| App corporativa / intranet | Tailwind CDN + shadcn-style manual + Lucide Icons |

**Regla de oro**: Tailwind siempre como base. Añadir encima según contexto.
Nunca mezclar Bootstrap con Tailwind.

## Paso 2: CDN imports — lo que se carga en cada archivo

### Base universal (todos los proyectos):
```html
<!-- Tailwind CSS -->
<script src="https://cdn.tailwindcss.com"></script>

<!-- Lucide Icons (SVG modernos) -->
<script src="https://unpkg.com/lucide@latest/dist/umd/lucide.min.js"></script>

<!-- AOS (animaciones al scroll) -->
<link rel="stylesheet" href="https://unpkg.com/aos@2.3.4/dist/aos.css"/>
<script src="https://unpkg.com/aos@2.3.4/dist/aos.js"></script>
<script>document.addEventListener('DOMContentLoaded', () => AOS.init({ once: true, duration: 600 }));</script>
```

### Añadir según stack elegido:

**DaisyUI** (para componentes rápidos y temas):
```html
<link href="https://cdn.jsdelivr.net/npm/daisyui@4/dist/full.min.css" rel="stylesheet"/>
```

**Flowbite** (componentes copy-paste):
```html
<link href="https://cdnjs.cloudflare.com/ajax/libs/flowbite/2.3.0/flowbite.min.css" rel="stylesheet"/>
<script src="https://cdnjs.cloudflare.com/ajax/libs/flowbite/2.3.0/flowbite.min.js"></script>
```

**GSAP** (animaciones avanzadas para landing/portfolio):
```html
<script src="https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.5/gsap.min.js"></script>
<script src="https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.5/ScrollTrigger.min.js"></script>
```

**Font Awesome** (si se necesitan más iconos):
```html
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.5.0/css/all.min.css"/>
```

## Paso 3: Sistema de diseño — definir ANTES de codear

Cada proyecto necesita tokens visuales consistentes. Definir en el `<style>` de cada archivo:

```css
:root {
  /* Paleta temática al dominio */
  --primary-50:  #f0f9ff;
  --primary-100: #e0f2fe;
  --primary-500: #0ea5e9;   /* color principal */
  --primary-600: #0284c7;   /* hover/active */
  --primary-700: #0369a1;   /* dark */

  /* Superficies */
  --bg:          #f8fafc;
  --surface:     #ffffff;
  --border:      #e2e8f0;

  /* Texto */
  --text:        #0f172a;
  --text-muted:  #64748b;

  /* Efectos */
  --radius-sm:   6px;
  --radius-md:   12px;
  --radius-lg:   20px;
  --shadow-sm:   0 1px 3px rgba(0,0,0,.08);
  --shadow-md:   0 4px 16px rgba(0,0,0,.10);
  --shadow-lg:   0 10px 40px rgba(0,0,0,.12);
}
```

### Paleta según dominio del producto:
- **Fintech / SaaS B2B**: Azul intenso o índigo — transmite confianza
- **Health / Bienestar**: Verde esmeralda o teal — calma y naturaleza  
- **Logística / Operaciones**: Naranja o ámbar — energía y urgencia
- **Agro / Medio ambiente**: Verde oscuro + tierra — orgánico, natural
- **Educación**: Violeta o azul periwinkle — creatividad y claridad
- **E-commerce / Retail**: Personalizar según marca del usuario
- **Portfolio / Creativo**: Monocromático + 1 acento fuerte

Nunca usar paleta genérica azul-gris si hay contexto de dominio disponible.

## Paso 4: Tipografía — reglas estrictas

Siempre importar desde Google Fonts. Nunca usar Inter, Arial, Roboto ni system fonts.

### Combinaciones por tipo de proyecto:

| Tipo | Display (headings) | Body |
|---|---|---|
| SaaS / Dashboard | Plus Jakarta Sans | Plus Jakarta Sans |
| Startup / Landing | Sora o Outfit | DM Sans |
| Portfolio creativo | Fraunces o Playfair | Instrument Sans |
| App corporativa | DM Sans | DM Sans |
| E-commerce | Nunito | Nunito Sans |
| App mobile-style | Public Sans | Public Sans |

```html
<!-- Ejemplo para SaaS -->
<link href="https://fonts.googleapis.com/css2?family=Plus+Jakarta+Sans:wght@400;500;600;700;800&display=swap" rel="stylesheet"/>

<style>
  body { font-family: 'Plus Jakarta Sans', sans-serif; }
</style>
```

**Escala tipográfica recomendada** (usar con Tailwind):
- H1: `text-4xl` o `text-5xl` + `font-bold` o `font-extrabold`
- H2: `text-2xl` o `text-3xl` + `font-semibold`
- H3: `text-xl` + `font-semibold`
- Body: `text-base` + `font-normal`
- Caption / muted: `text-sm` + `text-[var(--text-muted)]`

## Paso 6: Principios de diseño que SIEMPRE aplicar

### Espaciado generoso
- Padding de secciones: `py-20 px-6` o `py-24 px-8`
- Gap entre cards: `gap-6` mínimo, `gap-8` en desktop
- Padding interno de cards: `p-6` mínimo
- Nunca elementos apretados ni sin respiración

### Bordes redondeados modernos
- Cards: `rounded-2xl` (20px)
- Botones: `rounded-xl` o `rounded-2xl`
- Inputs: `rounded-xl`
- Badges / pills: `rounded-full`
- Evitar `rounded` (4px) — se ve antiguo

### Sombras sutiles y modernas
```css
/* Usar estas, no shadow-xl en todo */
shadow-sm   /* para cards base */
shadow-md   /* para elementos elevados */
hover:shadow-lg  /* en hover de cards */
shadow-blue-500/20  /* sombra con color — muy moderno */
```

### Hover states en todo elemento interactivo
```html
<!-- Siempre: transition + hover explícito -->
class="transition-all duration-200 hover:shadow-md hover:-translate-y-0.5"
```

### Gradientes modernos (no abusivos)
```css
/* Hero backgrounds */
bg-gradient-to-br from-slate-900 via-blue-950 to-slate-900

/* Texto con gradiente (sparingly) */
text-transparent bg-clip-text bg-gradient-to-r from-blue-500 to-cyan-400

/* Botones CTA */
bg-gradient-to-r from-blue-500 to-blue-600 hover:from-blue-400 hover:to-blue-500
```

### Efectos de fondo decorativos
```html
<!-- Blob/glow de fondo (hero sections) -->
<div class="absolute -top-40 -right-40 w-96 h-96 bg-blue-500/20 rounded-full blur-3xl pointer-events-none"></div>

<!-- Grid pattern -->
<div class="absolute inset-0 bg-[linear-gradient(rgba(0,0,0,.03)_1px,transparent_1px),linear-gradient(90deg,rgba(0,0,0,.03)_1px,transparent_1px)] bg-[size:32px_32px]"></div>
```

## Paso 7: Animaciones con AOS

Aplicar `data-aos` en elementos clave para que la página cobre vida al scroll:

```html
<!-- Básico -->
data-aos="fade-up"
data-aos="fade-in"
data-aos="zoom-in"

<!-- Con delay para secuencias (stagger) -->
<div data-aos="fade-up" data-aos-delay="0">Card 1</div>
<div data-aos="fade-up" data-aos-delay="100">Card 2</div>
<div data-aos="fade-up" data-aos-delay="200">Card 3</div>
```

**Regla**: aplicar a secciones y cards, no a cada elemento pequeño.

## Paso 8: Inicializar Lucide Icons

Al final del `<body>` de cada archivo, siempre añadir:

```html
<script>
  lucide.createIcons();
</script>
```

Usar íconos con: `<i data-lucide="nombre-icono" class="w-5 h-5"></i>`

Íconos de referencia más usados: `home`, `users`, `bar-chart-2`, `settings`,
`shopping-cart`, `bell`, `search`, `plus`, `arrow-right`, `check`, `x`,
`trending-up`, `layers`, `zap`, `shield`, `globe`, `mail`, `phone`

## Paso 10: Cuántas vistas generar

- **Mínimo**: 2 vistas (siempre — para mostrar navegación funcional)
- **Recomendado**: 3–4 vistas que cubran el flujo principal del producto
- **Máximo por entrega**: 5–6 archivos
- Si el usuario pide más: entregar el core primero, ofrecer continuar

Nomenclatura de archivos:
- Siempre `index.html` como punto de entrada
- Nombres descriptivos en minúsculas sin espacios: `pedidos.html`, `perfil.html`, `reportes.html`
- Nunca: `page2.html`, `view1.html`, `untitled.html`

## Paso 11: Checklist antes de entregar

- [ ] ¿Todas las vistas comparten la misma paleta y tipografía?
- [ ] ¿El ítem activo en el nav refleja correctamente la vista actual?
- [ ] ¿Los links entre páginas usan rutas relativas (no `href="#"`)?
- [ ] ¿Hay datos de ejemplo coherentes con el dominio (no "Lorem ipsum" en campos clave)?
- [ ] ¿Las animaciones AOS están aplicadas en secciones y cards?
- [ ] ¿`lucide.createIcons()` está al final del body en cada archivo?
- [ ] ¿Los hover states están en todos los elementos interactivos?
- [ ] ¿El diseño tiene suficiente espaciado (no se ve apretado)?
- [ ] ¿Abre correctamente en browser sin build steps?
